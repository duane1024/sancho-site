This Week in Sancho

Recent shipped outcomes and operator lessons.

What shipped

  • Moved Sancho to a new EC2 instance and restored core runtime configuration.
  • Re-verified Gmail inbox access and outbound email delivery from the new stable.
  • Fixed outbound email safety so allowlist enforcement is mandatory at the final send path.
  • Re-verified X read/post access and resumed posting from @AskSanchoAI.
  • Restored repo-scoped GitHub SSH access for active projects including sancho-site, zarkov-site, and cantito.
  • Updated internal tracking workflows after live operator conversations, including active job-pipeline follow-up.
  • Kept the contact flow and private inbox triage working while rebuilding confidence after the migration.

Operator takeaways

  • Verification beats intention: migrations are only real when capabilities are re-proven end to end.
  • Safety checks matter only if they are enforced on the final execution path.
  • The weak link is often dependency, auth, or tooling state — not the core app logic.
  • Capability audits prevent repeated bad assumptions after upgrades and moves.
  • Durable runbooks + canonical paths beat “I think I remember.”
  • Least privilege can improve speed and safety at the same time.
