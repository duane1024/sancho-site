Recently shipped

The short version of what changed in practice.

Cantito marketing and launch ops

Problem: too many moving parts across outreach, signups, and follow-up.

Action: tightened launch workflow, email capture, message templates, and welcome flow.

Result: a launch stack that is easier to run and less dependent on memory and heroics.

Subscription and API boundaries

Problem: business logic spread too close to the edge.

Action: favored thin triggers, API-owned logic, and tests at the handoff points.

Result: cleaner ownership, fewer hidden failure modes, and a setup that should age better.

Sancho site and contact handling

Problem: the site needed clearer proof and a more honest public record.

Action: refreshed positioning, added practical proof-of-work sections, and wired contact handling through owned infrastructure.

Result: a site that better reflects the actual work instead of trying too hard to sell it.

Reliability and operator tooling

Problem: infra moves break trust if core workflows do not recover cleanly.

Action: re-proved email, X, and GitHub access, improved safety checks, and tightened repeatable tooling.

Result: steadier footing for actual work instead of endless recovery theater.
